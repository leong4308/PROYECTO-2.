Definición

Un servidor web es un software que maneja solicitudes de clientes (generalmente navegadores web) y entrega recursos (como páginas web) en respuesta.

Función

Escucha:
Espera solicitudes en el puerto 80.
Recibe:
Captura la solicitud del cliente.
Busca:
Localiza el r ecurso en su sistema de archivos.
Envía:
Devuelve el recurso al cliente.

Tipos Principales

  • Apache HTTP Server
  • LiteSpeed Web Server
  • Nginx

Protocolo HTTP

HTTP es el protocolo para transferir documentos en la web, como HTML.

Características:
  • Sin Estado: Cada solicitud es independiente.
  • Cliente-Servidor: El cliente inicia la solicitud.
Métodos Comunes:
  • GET: Solicita un recurso.
  • POST: Envía datos al servidor.
  • PUT: Actualiza un recurso.
  • DELETE: Elimina un recurso.
Códigos de Estado:
  • 200 OK: Solicitud exitosa.
  • 404 Not Found: Recurso no encontrado.
  • 500 Internal Server Error: Error del servidor.
Versiones:
  • HTTP/1: Conexiones persistentes.
  • HTTP/2: Multiplexación y compresión.
  • HTTP/3: Basado en QUIC para mejor rendimiento.

Seguridad

Medidas Clave:
  • Configuración: Asegurar el servidor desactivando funciones innecesarias.
  • Control de Acceso: Implementar autenticación y autorización.
  • Monitoreo: Usar herramientas para detectar incidentes.
  • HTTPS: Cifrar comunicaciones con SSL/TLS.
  • Actualizaciones: Mantener software actualizado.
  • Protección contra Ataques: Usar firewalls y WAFs.

Escalabilidad

Horizontales:
  • Balanceo de Carga: Agregar más servidores para distribuir tráfico.
Verticales:
  • Recursos: Aumentar CPU, RAM y almacenamiento en servidores existentes.
Otras Estrategias:
  • Microservicios: Dividir aplicaciones en servicios independientes.
  • Bases de Datos Distribuidas: Escalar horizontalmente.
  • Caché Distribuido: Reducir carga en servidores y bases de datos.
